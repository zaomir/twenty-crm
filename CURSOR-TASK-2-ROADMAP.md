### 1.1 На VDS: Проверка 24/7 работы

**Требования:**
- [ ] Убедиться что контейнеры работают стабильно 24/7
- [ ] Проверить что health checks проходят
- [ ] Мониторить логи на ошибки
- [ ] Протестировать резервные копии (еженедельные)
- [ ] Мониторинг на месяц (утечки памяти, CPU, диск)

**Действия:**
```bash
# На VDS проверить ежедневно
docker compose ps
docker logs controlcenter-crm | grep -i error
docker stats controlcenter-crm
df -h /

# Еженедельно тестировать бэкап
docker exec controlcenter-postgres pg_dump \
  -U controlcenter_user controlcenter_prod > test_backup.sql
```

**Критерии готовности:**
- ✅ Нет ошибок в логах (неделю)
- ✅ Memory стабилен (< 500MB average)
- ✅ CPU < 10% average
- ✅ Диск не более 50% заполнен
- ✅ Health checks 100% успешны
